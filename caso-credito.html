Análisis de Crédito

En la banca minorista, decidir si aprobar un crédito rápido y de forma consistente es crítico: un análisis lento pierde clientes; uno inconsistente genera riesgos innecesarios. Este caso de uso muestra cómo un crew de agentes puede recuperar el perfil crediticio de un cliente por DNI, interpretar su score y factores de riesgo, enriquecer el análisis con contexto de mercado en tiempo real, generar una visualización del score y compilar un informe completo listo para el comité de crédito.

Vas a construir el crew en tres versiones: empezás con un único agente que ya resuelve el caso de negocio básico, y en cada iteración sumás capacidades hasta tener un crew de cinco agentes.

Studio

V1 — Básico 1 agente · Decisión crediticia ▶

Con un único agente ya podés resolver el problema de negocio central: dado un DNI, el agente consulta la base de datos de perfiles crediticios, interpreta el score, identifica los factores de riesgo y emite una recomendación de aprobación o rechazo. El secreto está en el backstory: sin él, el LLM trataría los datos que devuelve la herramienta como texto crudo. Con él, los interpreta con la voz y el criterio de un analista bancario experimentado.

Agente

Analista de Riesgo Crediticio

Role Analista Senior de Riesgo Crediticio
Goal Evaluar la solvencia del cliente con DNI {dni} para la solicitud {credit_request}, identificar los principales factores de riesgo y emitir una recomendación clara de otorgamiento de crédito
Backstory Sos un analista de crédito experimentado, especializado en banca minorista. Interpretás scores crediticios basados en modelos de machine learning, identificás señales de riesgo a partir del historial de pagos y la relación deuda-ingreso, y explicás decisiones financieras complejas de manera clara tanto para equipos de riesgo como para clientes.
Model gpt-5.4-mini
Tool teradata-credit-analysis

¿Por qué este Backstory? Ancla al LLM en la experiencia del dominio financiero y asegura que interprete lo que devuelve la herramienta —contexto del score, factores de riesgo, perfiles similares— con la voz correcta, no como datos crudos.

Tarea

Informe de Decisión Crediticia

Description Recuperá el perfil crediticio del DNI {dni}. Analizá el score, el nivel de riesgo y los principales factores de riesgo frente a esta solicitud: {credit_request}. Indicá si aprobarías el crédito y por qué.
Expected Output Un informe de decisión crediticia con: el score y la banda de riesgo, los 3 principales factores de riesgo, una recomendación de aprobación sí/no con justificación, y 2 a 3 perfiles de clientes comparables.
Agent Analista de Riesgo Crediticio

V2 — Explorador 3 agentes · Investigación web en vivo ▶

Un score alto suena excelente —pero ¿excelente en qué contexto? Si las tasas de crédito al consumo en Argentina están muy altas, incluso un perfil de bajo riesgo puede encontrar condiciones de mercado que compliquen el otorgamiento. En V2 sumás dos agentes nuevos: uno que va a buscar contexto de mercado en tiempo real usando Firecrawl, y otro que sintetiza los datos internos del cliente con esa inteligencia externa para producir una recomendación más robusta y fundamentada.

Esto también te muestra el primer handoff real entre agentes: el Analista de Riesgo le pasa su análisis al Investigador de Mercado, que lo usa como contexto para saber qué buscar, y ambos outputs llegan al Analista de Síntesis para el informe final.

Agregá estos dos agentes y tareas al crew de V1.

Agente

Investigador de Inteligencia de Mercado

Role Analista de Inteligencia de Mercado Financiero
Goal Investigar las condiciones actuales del mercado de crédito minorista en Argentina e identificar 2 a 3 tendencias relevantes para la solicitud {credit_request}
Backstory Sos un investigador financiero que monitorea el mercado de crédito minorista argentino. Seguís tendencias de tasas de interés, cambios regulatorios y datos de referencia para dar contexto a las decisiones crediticias individuales.
Model gpt-5.4-mini
Tool firecrawl

¿Por qué Firecrawl acá? Los datos internos te dan el score del cliente, pero no el contexto de mercado. El mismo perfil se ve diferente cuando las tasas, la inflación o la disponibilidad de crédito cambian. Firecrawl permite que un agente obtenga datos de mercado en tiempo real para enmarcar la decisión.

Tarea

Contexto de Mercado

Description Buscá las tasas de interés actuales del crédito minorista y las condiciones de disponibilidad de crédito en Argentina. Identificá 2 a 3 tendencias clave del mercado relevantes para esta solicitud: {credit_request}.
Expected Output Un briefing de mercado en 3 puntos: rango actual de tasas de referencia, tendencia clave que afecta el acceso al crédito, y un factor de riesgo en el entorno macroeconómico.
Agent Investigador de Inteligencia de Mercado
Agente

Analista de Síntesis Crediticia

Este tercer agente no usa ninguna herramienta —su valor es el razonamiento. Recibe el análisis del cliente y el contexto de mercado como inputs, y su trabajo es integrarlos en una recomendación única y bien fundamentada. Así funciona un comité de crédito real: una persona analiza el perfil, otra aporta contexto externo, y una tercera sintetiza y decide.

Role Analista de Síntesis de Decisiones Crediticias
Goal Combinar el análisis crediticio del cliente con DNI {dni} y el contexto de mercado actual para producir una recomendación completa sobre {credit_request}
Backstory Sos un analista de crédito senior que combina datos de riesgo internos con inteligencia de mercado para redactar recomendaciones crediticias equilibradas. Siempre contextualizás los scores individuales frente a las condiciones del mercado y comparaciones con perfiles similares.
Model gpt-5.4-mini
Tool ninguna — trabaja con el contexto de los dos agentes anteriores
Tarea

Informe Crediticio Integral

Description Combiná el análisis crediticio del cliente con DNI {dni} y el contexto de mercado anterior en un informe crediticio completo para {credit_request}. Incluí la decisión, el resumen de riesgo, la comparación con el mercado y la recomendación final.
Expected Output Un informe crediticio de una página: resumen del cliente, contexto del score, benchmarks de mercado, evaluación de riesgo y una recomendación final de aprobación o rechazo con justificación.
Agent Analista de Síntesis Crediticia

Ideas para probar — elegí un DNI y ajustá la solicitud

Pedí que contraste una solicitud de crédito con el contexto actual de tasas, inflación o disponibilidad de financiamiento.
Probá una solicitud conservadora y otra agresiva para el mismo DNI y pedí que explique cómo cambia la recomendación.
V3 — Crew Completo 5 agentes · Gráfico e informe ▶

En V3 el crew pasa de análisis a producción: dos agentes adicionales convierten los resultados en artefactos que podés llevar a una reunión de comité. El Visualizador genera un gráfico de barras que muestra dónde cae el score del cliente dentro de las cuatro bandas de riesgo — una imagen vale más que un número. El Redactor de Informes toma todos los outputs anteriores y los compila en un documento formal con resumen ejecutivo, listo para que cualquier decisor lo lea en 90 segundos.

Notá el cambio de modelo en el Visualizador: usamos gpt-5.4-nano porque generar un gráfico es una tarea liviana (formatear datos para una llamada a la herramienta). Reservamos gpt-5.4-mini para los agentes que necesitan razonar. Esa asignación inteligente de modelos es parte del diseño de un crew eficiente.

Agregá estos dos agentes y tareas sobre V2.

Agente

Visualizador de Datos Crediticios

Role Visualizador de Datos Crediticios
Goal Crear una visualización que muestre la posición del score del cliente con DNI {dni} dentro de las cuatro bandas de riesgo para apoyar la decisión sobre {credit_request}
Backstory Sos un especialista en visualización de datos que traduce métricas financieras en gráficos claros y listos para presentaciones ejecutivas. Elegís el tipo de gráfico que mejor comunica la historia que hay en los datos.
Model gpt-5.4-nano
Tool chart-generator

¿Por qué nano acá? La generación de gráficos es una tarea liviana —formatear datos para una llamada a la herramienta. No se necesita razonamiento pesado. gpt-5.4-nano es más rápido y económico para este paso.

Tarea

Visualización del Score

Description Generá un gráfico de barras que muestre el score crediticio del DNI {dni} en relación con las 4 bandas de riesgo: Muy Alto (menos de 520), Alto (520–619), Medio (620–719), Bajo (720 en adelante). Destacá la banda del cliente y conectá la visualización con {credit_request}.
Expected Output Una imagen de gráfico que muestre las bandas de distribución del score con la posición del cliente claramente marcada.
Agent Visualizador de Datos Crediticios
Agente

Redactor de Informes de Crédito

Role Redactor Senior de Informes de Crédito
Goal Compilar todos los hallazgos en un informe crediticio formal para {report_audience}, usando el DNI {dni} y la solicitud {credit_request}
Backstory Sos un oficial de crédito senior que produce informes formales para comités de crédito. Tus documentos son concisos, bien estructurados y orientados a la toma de decisiones. Combinás el scoring cuantitativo con el juicio cualitativo y siempre abrís con un resumen ejecutivo de 3 oraciones.
Model gpt-5.4-mini
Tool ninguna — compila el output de todos los agentes anteriores
Tarea

Informe Crediticio Completo

Description Redactá un informe completo para {report_audience} sobre el DNI {dni} y la solicitud {credit_request}. Incorporá el análisis crediticio, el contexto de mercado, la descripción de la visualización del score y un resumen ejecutivo de 3 oraciones al inicio. Formatealo como un documento formal con secciones.
Expected Output Un informe crediticio completo: resumen ejecutivo, perfil del cliente, análisis del score, contexto de mercado, evaluación de riesgo y decisión de aprobación o rechazo con fundamento.
Agent Redactor de Informes de Crédito
