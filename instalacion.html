Instalação (código)

Código · Python

O que você vai aprender

Este é o guia do caminho de código. Se preferir trabalhar sem instalar nada, o caminho alternativo é o Studio — coberto em Começar.

Ao terminar este guia você terá:

  • uv instalado como gerenciador de ambientes e pacotes
  • O CLI do CrewAI instalado e verificado
  • Um projeto CrewAI rodando localmente
  • As ferramentas do MongoDB disponíveis no seu ambiente

Antes de começar

Você precisa ter instalados:

RequisitoDetalhe
Python 3.10–3.13 Versões anteriores não são compatíveis com o CrewAI. Verifique com python3 --version.
Editor de código VS Code (recomendado) ou qualquer editor que você use habitualmente.
Terminal Bash / Zsh (macOS ou Linux) ou PowerShell (Windows).
API key do OpenAI Sua chave pessoal. Você vai precisar dela no Passo 5.

Passo 1 — Instalar uv

uv é o gerenciador de ambientes e pacotes que o CrewAI utiliza. Instale conforme o seu sistema operacional:

macOS / Linux

bash 
curl -LsSf https://astral.sh/uv/install.sh | sh

Windows (PowerShell)

powershell 
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Alternativa com Homebrew (macOS)

bash 
brew install uv

Após a instalação, verifique se está funcionando:

bash 
uv --version
Ponto de controle: o comando acima deve exibir uma versão (ex. uv 0.x.x). Se não for encontrado, reinicie o terminal para que o PATH seja atualizado.

Passo 2 — Instalar o CLI do CrewAI

Instale o CLI do CrewAI como ferramenta global com uv tool:

bash 
uv tool install crewai==1.14.7

Verifique a instalação:

bash 
crewai version
Ponto de controle: você deve ver crewai, version 1.14.7. Se o comando não for encontrado, feche e reabra o terminal.

Passo 3 — Criar o projeto

Use o CLI para gerar a estrutura do projeto:

bash 
crewai create crew mi_projeto

O CLI vai perguntar o provedor e o modelo de LLM. Responda assim:

Pergunta do CLIResposta recomendada
Selecione o provedor de LLM openai ou anthropic
Selecione o modelo gpt-5.4-mini (OpenAI) ou claude-haiku-4-5 (Anthropic)

Isso gera uma pasta mi_projeto/ com a seguinte estrutura:

bash 
mi_projeto/
├── src/mi_projeto/
│   ├── config/
│   │   ├── agents.yaml     # Definição de agentes
│   │   └── tasks.yaml      # Definição de tarefas
│   ├── crew.py             # Classe do crew
│   └── main.py             # Ponto de entrada
├── pyproject.toml
└── .env                    # Variáveis de ambiente (a preencher)
Ponto de controle: a pasta mi_projeto/ existe e contém os arquivos listados acima.

Passo 4 — Instalar dependências

Entre no diretório do projeto e instale as dependências:

bash 
cd mi_projeto
crewai install

Este comando cria um ambiente virtual isolado e baixa todos os pacotes declarados em pyproject.toml. Pode levar alguns minutos na primeira vez.

Ponto de controle: a instalação finaliza sem erros. Você verá um resumo dos pacotes instalados no terminal.

Passo 5 — Configurar o arquivo .env

Abra o arquivo .env na raiz do seu projeto e adicione suas credenciais:

env 
OPENAI_API_KEY=sk-...
MONGODB_CONNECTION_STRING=mongodb+srv://...
VariávelValor
OPENAI_API_KEY Sua chave pessoal do OpenAI
MONGODB_CONNECTION_STRING Fornecida no evento (necessária para as ferramentas do MongoDB)
Importante: nunca envie o arquivo .env para um repositório público. Verifique se o .gitignore o inclui.

Passo 6 — Executar o crew

Execute o crew de exemplo gerado pelo CLI:

bash 
crewai run

Você verá os agentes pensando, usando ferramentas e passando resultados no seu terminal. Isso pode levar alguns minutos — é normal, os agentes estão consultando o LLM a cada passo.

Ponto de controle: o crew finaliza e exibe um resultado no terminal. A partir daqui você pode modificar agents.yaml, tasks.yaml e crew.py para construir seus próprios crews.

Flows — Pipelines deterministas

Os Flows são para quando você quer que os passos sejam executados em uma ordem garantida com estado tipado compartilhado. Use um Flow quando do contrário encadearia scripts manualmente ou precisar passar dados tipados entre etapas.

Crew vs Flow: a comparação conceitual está em Começar → Crew vs Flow. Os percursos principais do workshop usam Crews; os Flows são uma opção avançada para pipelines personalizados.

Estrutura mínima de um Flow:

python 
from crewai.flow.flow import Flow, start, listen
from pydantic import BaseModel

class ClaimsState(BaseModel):
    case_id: str = "REC-001"
    classification: str = ""
    resolution: str = ""

class ClaimsAnalysisFlow(Flow[ClaimsState]):
    @start()
    def classify_claim(self):
        # Lógica de classificação (agente ou chamada direta à ferramenta)
        self.state.classification = "..."   # salvar no estado tipado

    @listen(classify_claim)
    def draft_resolution(self):
        # Recebe o output de classify_claim; usa self.state.classification
        self.state.resolution = "..."

flow = ClaimsAnalysisFlow()
result = flow.kickoff()
print(result)
  • @start() marca o ponto de entrada do Flow
  • @listen(step_name) é executado após aquele passo ser concluído
  • Todos os passos compartilham self.state — um modelo Pydantic tipado que você define
  • Você pode incorporar um Crew completo como um passo dentro de um Flow
  • Os Flows são apenas código — não há suporte no Studio

Instalar as ferramentas do MongoDB

Uma vez que seu projeto local esteja rodando, você adiciona as ferramentas do MongoDB para se conectar com os datasets do evento. Instale os pacotes necessários com pip:

bash 
pip install mongodb-credit-analysis==0.1.3
pip install mongodb-conversational-bi==0.1.4
pip install mongodb-claims-analysis==0.1.3
pip install mongodb-vector-search==0.1.0
PacoteCaso de uso
mongodb-credit-analysis==0.1.3 Análise de Crédito — recupera perfis de crédito por CPF
mongodb-conversational-bi==0.1.4 BI Conversacional — consultas NLP sobre produtos e vendas
mongodb-claims-analysis==0.1.3 Reclamações por Caso e Reclamações RAG — recupera e busca reclamações semanticamente
mongodb-vector-search==0.1.0 Ferramenta genérica — acesso livre a todos os datasets nos modos factual, semântico, híbrido ou auto

Dica: Essas mesmas ferramentas estão disponíveis no AMP/Studio para o caminho sem código. No Studio você as busca em Browse AMP no painel de ferramentas do seu agente, sem precisar instalar nada.

Importe-as no seu código assim:

python 
from mongodb_credit_analysis import MongoDBCreditAnalysis
from mongodb_conversational_bi import MongoDBConversationalBI
from mongodb_claims_analysis import MongoDBClaimsAnalysis
from mongodb_vector_search import MongoDBVectorSearch

Linha de chegada

Se você chegou até aqui, seu ambiente está pronto. Verifique se tudo está em ordem antes de passar ao primeiro caso de uso:

  • uv instalado e uv --version responde corretamente
  • CLI do CrewAI instalado — crewai version exibe 1.14.7
  • Projeto criado com crewai create crew mi_projeto
  • Dependências instaladas com crewai install sem erros
  • Arquivo .env configurado com OPENAI_API_KEY e MONGODB_CONNECTION_STRING
  • crewai run finaliza com êxito e exibe um resultado
  • Ferramentas do MongoDB instaladas com pip install
