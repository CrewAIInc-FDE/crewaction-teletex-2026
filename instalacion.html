Instalación (código)

Código · Python

Qué vas a aprender

Esta es la guía del camino de código. Si preferís trabajar sin instalar nada, el camino alternativo es Studio — cubierto en Comenzar.

Al terminar esta guía vas a tener:

  • uv instalado como gestor de entornos y paquetes
  • El CLI de CrewAI instalado y verificado
  • Un proyecto CrewAI corriendo localmente
  • Las herramientas de Teradata disponibles en tu entorno

Antes de empezar

Necesitás tener instalados:

RequisitoDetalle
Python 3.10–3.13 Versiones anteriores no son compatibles con CrewAI. Verificá con python3 --version.
Editor de código VS Code (recomendado) o cualquier editor que uses habitualmente.
Terminal Bash / Zsh (macOS o Linux) o PowerShell (Windows).
API key de OpenAI Tu clave personal. La vas a necesitar en el Paso 5.

Paso 1 — Instalar uv

uv es el gestor de entornos y paquetes que usa CrewAI. Instalalo según tu sistema operativo:

macOS / Linux

bash 
curl -LsSf https://astral.sh/uv/install.sh | sh

Windows (PowerShell)

powershell 
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Alternativa con Homebrew (macOS)

bash 
brew install uv

Una vez instalado, verificá que funcione:

bash 
uv --version
Punto de control: el comando anterior debe mostrar una versión (ej. uv 0.x.x). Si no lo encontrás, reiniciá tu terminal para que el PATH se actualice.

Paso 2 — Instalar el CLI de CrewAI

Instalá el CLI de CrewAI como herramienta global con uv tool:

bash 
uv tool install crewai==1.14.7

Verificá la instalación:

bash 
crewai version
Punto de control: deberías ver crewai, version 1.14.7. Si el comando no se encuentra, cerrá y volvé a abrir la terminal.

Paso 3 — Crear el proyecto

Usá el CLI para generar la estructura del proyecto:

bash 
crewai create crew mi_proyecto

El CLI va a preguntarte el proveedor y modelo de LLM. Respondé así:

Pregunta del CLIRespuesta recomendada
Seleccioná el proveedor de LLM openai (opción por defecto)
Seleccioná el modelo gpt-5.4-mini

Esto genera una carpeta mi_proyecto/ con la siguiente estructura:

bash 
mi_proyecto/
├── src/mi_proyecto/
│   ├── config/
│   │   ├── agents.yaml     # Definición de agentes
│   │   └── tasks.yaml      # Definición de tareas
│   ├── crew.py             # Clase del crew
│   └── main.py             # Punto de entrada
├── pyproject.toml
└── .env                    # Variables de entorno (a completar)
Punto de control: la carpeta mi_proyecto/ existe y contiene los archivos listados arriba.

Paso 4 — Instalar dependencias

Entrá al directorio del proyecto e instalá las dependencias:

bash 
cd mi_proyecto
crewai install

Este comando crea un entorno virtual aislado y descarga todos los paquetes declarados en pyproject.toml. Puede tardar un par de minutos la primera vez.

Punto de control: la instalación finaliza sin errores. Vas a ver un resumen de paquetes instalados en la terminal.

Paso 5 — Configurar el archivo .env

Abrí el archivo .env en la raíz de tu proyecto y agregá tus credenciales:

env 
OPENAI_API_KEY=sk-...
TERADATA_STORE_CONNECTION_STRING=mongodb+srv://...
VariableValor
OPENAI_API_KEY Tu clave personal de OpenAI
TERADATA_STORE_CONNECTION_STRING Se proporciona en el evento (necesaria para las herramientas de Teradata)
Importante: nunca subas el archivo .env a un repositorio público. Verificá que .gitignore lo incluya.

Paso 6 — Correr el crew

Ejecutá el crew de ejemplo que generó el CLI:

bash 
crewai run

Vas a ver a los agentes pensando, usando herramientas y pasando resultados en tu terminal. Esto puede tardar unos minutos — es normal, los agentes están consultando el LLM en cada paso.

Punto de control: el crew finaliza y muestra un resultado en la terminal. A partir de acá podés modificar agents.yaml, tasks.yaml y crew.py para construir tus propios crews.

Flows — Pipelines deterministas

Los Flows son para cuando querés que los pasos se ejecuten en un orden garantizado con estado tipado compartido. Usá un Flow cuando de otro modo encadenarías scripts manualmente o necesitás pasar datos tipados entre etapas.

Crew vs Flow: la comparación conceptual está en Comenzar → Crew vs Flow. Los tres casos de uso del taller usan Crews; los Flows son una opción avanzada para pipelines personalizados.

Estructura mínima de un Flow:

python 
from crewai.flow.flow import Flow, start, listen
from pydantic import BaseModel

class ClaimsState(BaseModel):
    case_id: str = "REC-001"
    classification: str = ""
    resolution: str = ""

class ClaimsAnalysisFlow(Flow[ClaimsState]):
    @start()
    def classify_claim(self):
        # Lógica de clasificación (agente o llamada directa a herramienta)
        self.state.classification = "..."   # guardar en estado tipado

    @listen(classify_claim)
    def draft_resolution(self):
        # Recibe el output de classify_claim; usa self.state.classification
        self.state.resolution = "..."

flow = ClaimsAnalysisFlow()
result = flow.kickoff()
print(result)
  • @start() marca el punto de entrada del Flow
  • @listen(step_name) se ejecuta después de que ese paso completa
  • Todos los pasos comparten self.state — un modelo Pydantic tipado que vos definís
  • Podés embeber un Crew completo como un paso dentro de un Flow
  • Los Flows son solo código — no hay soporte en Studio

Instalar las herramientas de Teradata

Una vez que tu proyecto local corre, sumás las herramientas de Teradata para conectarte con los datasets del evento. Instalá los paquetes que necesites con pip:

bash 
pip install teradata-credit-analysis==0.1.3
pip install teradata-conversational-bi==0.1.4
pip install teradata-claims-analysis==0.1.3
pip install teradata-store-search==0.1.0
PaqueteCaso de uso
teradata-credit-analysis==0.1.3 Análisis de Crédito — recupera perfiles crediticios por DNI
teradata-conversational-bi==0.1.4 BI Conversacional — consultas NLP sobre productos y ventas
teradata-claims-analysis==0.1.3 Análisis de Reclamos — recupera y busca reclamos semánticamente
teradata-store-search==0.1.0 Herramienta genérica — acceso libre a todos los datasets en modos factual, semántico, híbrido o auto

Tip: Estas mismas herramientas están disponibles en AMP/Studio para el camino sin código. En Studio las buscás desde Browse AMP en el panel de herramientas de tu agente, sin necesidad de instalar nada.

Importalas en tu código así:

python 
from teradata_credit_analysis import TeradataCreditAnalysis
from teradata_conversational_bi import TeradataConversationalBI
from teradata_claims_analysis import TeradataClaimsAnalysis
from teradata_store_search import TeradataStoreSearch

Línea de llegada

Si llegaste hasta acá, tu entorno está listo. Verificá que todo esté en orden antes de pasar al primer caso de uso:

  • uv instalado y uv --version responde correctamente
  • CLI de CrewAI instalado — crewai version muestra 1.14.7
  • Proyecto creado con crewai create crew mi_proyecto
  • Dependencias instaladas con crewai install sin errores
  • Archivo .env configurado con OPENAI_API_KEY y TERADATA_STORE_CONNECTION_STRING
  • crewai run finaliza exitosamente y muestra un resultado
  • Herramientas de Teradata instaladas con pip install
