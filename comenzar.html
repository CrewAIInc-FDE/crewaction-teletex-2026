Começar

Studio · Sem código

Pré-requisitos

Usuários do Studio — você não precisa configurar nada. As variáveis de ambiente e as credenciais das ferramentas estão pré-configuradas no AMP. Você pode continuar lendo para conhecer o MongoDB antes de montar seu primeiro Crew.

Prefere trabalhar com código? Então Instalação é o seu próximo passo — lá você encontra o setup completo com uv, o CLI do CrewAI, e as ferramentas do MongoDB instaláveis com pip.

Modelos recomendados

Hoje temos disponíveis quatro modelos — experimente cada um e observe as diferenças de velocidade, custo e qualidade de resposta:

ModeloProvedorIdeal para
gpt-5.4-mini OpenAI A maioria dos agentes — recuperação, explicação, análise, síntese, geração de relatórios
gpt-5.4-nano OpenAI Agentes leves — classificação simples, resumos breves, gráficos em markdown
claude-haiku-4-5 Anthropic Respostas rápidas e concisas, tarefas estruturadas, baixo custo
claude-sonnet-4-6 Anthropic Raciocínio complexo, escrita elaborada, análise detalhada

Por onde começar: use gpt-5.4-mini como ponto de partida para a maioria dos agentes. Para tarefas simples de ação única, prefira gpt-5.4-nano ou claude-haiku-4-5. Quando precisar de raciocínio mais profundo ou respostas mais elaboradas, experimente claude-sonnet-4-6. Teste os quatro e compare — velocidade, custo e qualidade variam bastante.

Dica: Você não interage diretamente com essas coleções — as ferramentas AMP gerenciam todas as consultas. A tabela acima é apenas contexto para entender quais dados sustentam cada caso de uso.

Crew vs Flow

CrewFlow
O que é Uma equipe de agentes de IA que colaboram em tarefas Um pipeline determinístico de etapas (pode incluir agentes)
Estado Passado entre agentes por meio do contexto de tarefas Objeto de estado Pydantic tipado (self.state)
Quando usar Quando múltiplos agentes precisam raciocinar juntos e fazer handoffs Quando as etapas devem ser executadas em uma ordem fixa e previsível
Disponível no Studio? Sim Não — somente código
Exemplo Analista de crédito → pesquisador de mercado → redator de relatório Carregar PDFs → fragmentar → incorporar → armazenar

Para este workshop: os percursos principais usam Crews. Tanto o caminho do Studio quanto o de Código criam Crews. Os Flows são apresentados em Instalação como opção avançada para o caminho de código.

Seu primeiro Crew

Este é o percurso genérico no Studio. Cada seção de caso de uso mais abaixo tem campos prontos para copiar e adaptar.

Studio: passo a passo

  1. 1

    Criar um novo Crew

    Abra o CrewAI Studio e clique em New Crew. Dê o nome que preferir para reconhecê-lo depois.

  2. 2

    Adicionar um Agente

    Clique em Add Agent. Preencha quatro campos:

    CampoO que faz
    RoleDefine a especialidade do agente. Seja específico — "Senior Credit Risk Analyst" funciona melhor do que "Analyst".
    GoalO que este agente deve alcançar. Inclua inputs entre chaves, como {cpf} ou {claim_text}, para que o Studio os solicite ao executar.
    BackstoryContexto que molda a voz e o julgamento do agente. Aqui você incorpora o conhecimento do domínio.
    ModelO LLM que este agente usa. Use gpt-5.4-mini como padrão.
  3. 3

    Adicionar uma Ferramenta do AMP

    Na seção de ferramentas do agente, clique em Add Tool → Browse AMP. Busque pelo nome da ferramenta (ex.: mongodb-credit-analysis). Selecione-a e adicione ao agente.

    Dica: firecrawl e mongodb-vector-search também estão disponíveis no AMP para crews V2/V3 e explorações personalizadas.

  4. 4

    Adicionar uma Tarefa

    Clique em Add Task. Preencha:

    CampoO que faz
    DescriptionO que o agente deve fazer. Use os mesmos inputs entre chaves que aparecem no Goal.
    Expected OutputO formato exato que você quer receber. Quanto mais específico, melhor o resultado.
    AgentAtribua esta tarefa ao agente correto.
  5. 5

    Executar

    Clique em Run. Observe como os agentes trabalham. O resultado de cada agente é alimentado automaticamente para a tarefa seguinte.
